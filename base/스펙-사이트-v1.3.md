# 좋은비전 사이트 설계 스펙 v1.3 (확정본)

작성일: 2026-07-17 · 상태: 확정 (Claude Code 핸드오프 동반)
대체: v1(BaseLayout 오리엔테이션 스펙), v1.2(통합 스펙)를 이 문서로 흡수. 이것이 정본.
검증 기준: 센스리더(주, 브라우저 탭키 환경) + NVDA(부) 교집합.
제1 목표: **센스리더 사용자가 Tab을 덜 누르고 원하는 정보에 바로 닿는다.**

---

## 0. 정체성과 구조 (한 장 요약)

### 0.1 정체성 선언

**좋은비전 사이트 = 레퍼럴 허브.** 데이터를 보관하지 않고 연결한다.
- 기관의 성격(연결 기관) = 사이트의 구조(환승역) = 센터장의 검색 전략(앵커 방사). 세 층이 같은 패턴.
- 무보관 원칙: 이용자 데이터를 사이트에 저장하지 않는다. 유일한 보관 예외 = 법적 의무 문서(기부금 사용내역 PDF).
- 판정 기준(향후 기능 요청 시): "그거 보관인가 연결인가?" 보관이면 기본 거절.
- 우리가 원본인 콘텐츠(활동소식·칼럼)는 발행이지 보관이 아님 — 허용.

### 0.2 사용자 2축

```
축 A. 당사자/가족 — 대부분 대리 행위자(가족·지인)
  근거: 갓 장애가 생긴 당사자는 에너지가 없다. 움직이는 건 주변인.
  실측 검색어: "중도시각장애인", "중도장애인" (이미 센터장이 랭크됨)
  진입: 검색 / 홈 / 처음 오신 분께(예약 상태)
  종료 상태: 전화 (tel:)
  문체: 조력자 대상. 판정·등록을 첫 화면에서 들이밀지 않는다("벽이 큼").
  전화 문구(전 페이지): "본인이 아니어도 됩니다. 가족·지인의 상담 문의를 환영합니다."

축 B. 외부 지원자 — 기업·학교·기관 담당자 (대부분 비시각)
  진입: 검색 → 개별 페이지 직착지 (홈 아님)
  종료 상태: 강연 폼(예약 상태) / 후원 계좌
```

### 0.3 앵커 방사 (인터뷰 반영)

센터장 이름 = 지역 생태계의 검색 기준점. 사이트의 일 = 그 앵커에 착지한 사람을 협력기관으로 방사.
센터장 상담 = Fit 매칭(상황 → 기관). 사이트는 그중 **반복 패턴**을 전사해 24시간 처리하고, 미묘한 판단이 필요한 케이스만 전화로 흐르게 한다. 기관 자산 = 네트워크 · 경험 · 전문상담능력 · 센터장 브랜드네임.

## 1. 상태 구조 (오토마타)

```
홈 ────────────────── 허브
├─ 기관소개 ─────────── 내용 (첫 줄: "어디로 가야 할지 알려드립니다")
│   └─ 센터장 소개 ───── 내용 [P0 승격 — 이름 검색 유입의 착지 페이지]
├─ 상담안내 ─────────── 내용 → 말단: 전화
├─ 후원안내 ─────────── 내용 → 말단: 계좌
│   └─ 기부금 사용내역 ── 문서고 → 말단: PDF (연도별, 관리자 업로드)
├─ 소식 ─────────────── 목록 (태그 필터)
│   ├─ [url 있는 글] ──── 외부 직행 말단 (원본이 바깥)
│   └─ [url 없는 글] ──── 내부 상세 (우리가 원본 = 활동소식)
├─ 협력기관 ─────────── footer 상시 그리드 → 각 로고 = 태그 페이지로
└─ /소식/태그/{기관}/ ── 자동 생성 (방사의 실행 지점)

예약 상태 (라벨 동결은 항목 추가를 막지 않음 — 추후 append):
처음 오신 분께 · 강연 안내 · 칼럼(columns)
```

**주 메뉴 4개: 기관소개 · 상담안내 · 후원안내 · 소식**
header 상시: 검색창 + 전화 / footer 상시: 협력기관 그리드 + 연락처

## 2. 탐색 계약 — "Tab을 덜 누르게" (제1 목표의 실행)

### 2.1 원리
- **5키 원리**: 기준점은 위치가 아니라 "어디서든 통하는 같은 첫 동작". 보조장치는 더하지 않고 뺀다.
- **2단 탐색**: 순간이동(F2/H/1~6, 전역) → Tab(지역, 도착한 방 안에서만). Tab 전수조사가 필요한 구조 자체를 없앤다.
- **전용 단축키 제작 금지**: 범용 키가 유일한 목적지를 갖게 만들어 사실상 전용키화.
- **교집합 골격**: 두 리더에서 동일한 H, 1~6, Tab만 1차 문법. 랜드마크(센스리더 J/NVDA D)는 2차.

### 2.2 문서 순서 (전 페이지 고정, 배포 후 동결)

```
1. <header>
   ① role="search" 검색창 — 페이지 유일의 편집창
      레이블: "검색 — 현재 위치: {페이지명}"
      → F2 계열(브라우저 탭키 F2 / 가상커서 Ctrl+F2 / NVDA E) 한 번 = 검색창.
        키가 아니라 편집창의 유일성이 계약.
   ② 전화 링크: <a href="tel:...">전화 상담: 0XX-XXX-XXXX</a>
      번호를 텍스트로 노출. 불변식: **F2 → Tab 1회 = 전화**
2. <nav aria-label="주 메뉴"> — 4항목, 라벨 동결
3. <main>
   <h1>{페이지명}</h1>
   {breadcrumb: 텍스트+링크, h1 직하}
   {본문 — h2 섹션}
   <h2>여기서 갈 수 있는 곳</h2> — 전 페이지 마지막 h2, 표제 동결
4. <footer> — 협력기관 그리드(정적, 회전 금지) + 연락처
```

### 2.3 헤딩 위계 (1차 문법)
h1 = 페이지명(1개, 현재 위치 겸). h2 = 섹션 + 마지막은 항상 "여기서 갈 수 있는 곳". h3 = 필요시.
검증: 숫자 2 반복만으로 페이지 아웃라인이 잡혀야 한다.

### 2.4 행동 말단 규칙
사이트 내 폼·단계·위저드 금지. 전이 1회로 종료.
상담=전화(tel:+번호 텍스트) / 후원=계좌 텍스트+복사 버튼 / (예약: 강연=외부 폼).
→ 검색창 외 입력창이 존재할 이유가 없음 = 편집창 유일성 구조적 보장.
신규 기능 판정 질문: "편집창 유일성을 깨는가?" + "보관인가 연결인가?"

### 2.5 링크 텍스트 자족성 (링크 목록 ;키 대응)
링크는 단독으로 읽혀도 완결: 외부 소식 "[{org}] {title} (외부 사이트, 새 탭에서 열림)" — org 태그에서 템플릿 자동 조립, 편집자 수기 금지. PDF는 "{연도} 기부금 사용내역 (PDF)". "(새 탭에서 열림)"은 a 태그 안.

### 2.6 랜드마크 (2차, 4개 고정)
search / nav(주 메뉴) / main / footer. 추가 금지.

### 2.7 첫 줄 계약 (간략 읽기 모드)
title: `{페이지명} – 좋은비전` (페이지명 선행). h1 = title의 페이지명과 동일 문자열.

### 2.8 라벨 동결
센스리더 문자값 등록(URL별 단축 저장) 보호. 동결 대상: 주 메뉴 항목명, "여기서 갈 수 있는 곳", 전화·후원 행동 라벨. 변경 시 base/ 판정 문서 기록 후 일괄 변경.

### 2.9 동적 콘텐츠 배치
센스리더 마크(Ctrl+M) = 라인 번호 기반. header~주 메뉴 구간 정적 동결. 자동 갱신 콘텐츠는 본문 하단. 예외: 소식 목록의 태그 필터 행은 h1 직하 허용(목록 자체가 동적이므로 실질 피해 없음).

### 2.10 DOM 순서 통일
DOM 순서 = 읽기 순서 = 시각 순서. CSS로 시각 순서만 바꾸는 것 금지.

## 3. 소식 시스템 (태그 = 필터 = 방사 장치)

### 3.1 스키마 (notices 컬렉션)
```yaml
title: string
date: date
org: [기관명 태그]     # 좋은비전 = 우리 활동. url 붙이는 순간 아는 정보 = 입력비용 0
url: string?           # 있으면 외부 직행, 없으면 내부 상세 — 필드가 아니라 데이터에서 도출
body: markdown?        # url 없을 때만. 이미지 alt 필수(CMS 위젯 강제)
```
편집자 규칙 한 문장: "남의 소식이면 주소를 붙이고, 우리 일이면 내용을 쓰세요."

### 3.2 태그의 이중 역할
- 편집자에겐: 사실 기입 (분류 판단 없음 — 주제 분류는 하지 않는다, 유지비 0 우선)
- 사용자에겐: 필터 (관심 기관만 골라 보기 — 기관은 내용 종류의 대리 지표)

### 3.3 필터 구현
- 소식 목록 상단(h1 직하)에 태그 링크 행: [전체] [좋은비전] [○○복지관] ...
- **정적 링크만.** JS 동적 필터링 금지(무음 화면 변경은 스크린리더에 사고). 각 태그 = `/소식/태그/{org}/` 정적 페이지. 새 h1("소식 — {org}")이 필터 결과를 확언.
- 필터 행 노출은 글 2건 이상 기관만 (태그 남발 = 알림판 방지).
- 태그 페이지 URL = 북마크 가능한 정적 주소 → "관심 기관 구독"을 사용자 측 브라우저 북마크로 해결 (무보관 유지).
- 태그 어휘 = 기관명 authority (표기 흔들림은 태그 목록 관리 한 곳에서).

### 3.4 태그 페이지 = 방사 페이지
글 목록만으론 매칭 불가. 상단에 매칭 단락 필수 (partners 컬렉션에서 주입):
```yaml
partners: { name, url, logo?, desc(두 문장), when("이런 때 가세요" 한 줄) }
```
when 어휘 출처 = 센터장이 상담에서 반복하는 "어디 가세요" 답변 (다음 인터뷰 질문: "제일 자주 하는 답변 5개").

### 3.5 소식지 구독
"매달 받고 싶다" 요청은 사이트 밖(카톡 채널/전화)으로. 소식 페이지에 안내 한 줄. 수집은 기존 판정대로 외부(구글폼 등). 저작권상 콘텐츠 원문 보관 금지 — 소식(링크)만 준다.

## 4. 전이표 방침 ("여기서 갈 수 있는 곳")

| 페이지 | 항상 | 추가 |
|---|---|---|
| 기관소개 | 전화 | 센터장 소개, 소식 |
| 센터장 소개 | **전화** | 협력기관 태그 페이지들(방사), 활동소식 |
| 상담안내 | **전화** | 기관소개 |
| 후원안내 | **계좌** | /소식/태그/좋은비전(활동 실증), 기부금 내역 PDF |
| 소식 목록/상세 | 전화 | 후원안내, 해당 org 태그 페이지 |
| 태그 페이지 | 해당 기관 외부 링크 | 전화, 다른 태그 |

원칙: **막다른 페이지 0개.** 모든 페이지가 전화 또는 계좌(추후 +강연 폼)로 흐른다.
후원 신뢰의 이중 근거: 기부금 PDF(숫자) + 활동소식(이야기).

## 6. 콘텐츠 규약 (지금은 그릇만, 글은 나중)

- columns 컬렉션은 **예약**: 스키마만 확정 — {title, author, date, audience: 당사자가족|외부지원자, situations: []}. audience가 전이표 출구 자동 분기.
- 글 작성 규약 2줄 (전자책 투영 대비, 시행은 글 시작 시): ① 각 글 자기완결(사이트 맥락 의존 금지) ② 구조는 헤딩으로만.
- 전자책(HTML)은 §8 로드맵 — 트리거: 칼럼 10편 축적 시. 같은 원본의 빌드 타겟 추가일 뿐.
- situations 어휘 시드(인터뷰에서 수집, 추후 사용): 책 읽고 싶다 / 우리 지역 기관을 모른다 / 집에만 있다(역량 있는데) / 동호회·사람이 그립다.
