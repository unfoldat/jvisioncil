## 2026-07-21 전면 개편 — 독수리 검토 목업 이식(좋은비전 CIL 디자인) 재판정

기존 프로토타입(스펙 v1.3 기준: 검색창 1개뿐인 편집창, "여기서 갈 수 있는 곳" 전이표 고정,
랜드마크 1개 초과 금지, 협력기관 그리드 footer 전용)을 독수리가 검토·승인한 최종 디자인으로
교체하면서, 위 계약 중 상당수가 구조적으로 바뀌었다. 새 계약을 `scripts/verify.mjs`(D1~D5 →
E1~E7로 재작성)에 반영했고, 그 기준으로 아래를 재판정한다.

- **5.1.1(대체 텍스트)**: 조치완료 유지. 새로 추가된 카드형 이미지(홈 최근 소식은 이미지 없음,
  "함께 만드는 변화" 카드만 이미지 사용)도 기존 `image_alt` 필수 규칙 그대로.
- **5.3.1(표의 구성)**: **해당없음 → 조치완료로 변경.** 소식 목록/태그 페이지가 데스크톱에서
  `<table>`(기관·제목·날짜, `<th scope="col">` 사용)로 렌더링된다 — 처음 도입. 좁은 화면·글자
  배율 확대(`html[data-zoom="wide"]`) 시에는 같은 데이터를 `<article>` 목록(`.mobile-list`)으로
  전환해, 작은 화면에서 가로 스크롤 표 대신 읽기 쉬운 카드로 보여준다.
- **6.1.1/6.1.2(키보드 사용·초점)**: 새 인터랙션(글자 배율 슬라이더, 모바일 메뉴 토글 버튼)
  전부 순수 HTML `<input type=range>`/`<button>` — 키보드로 전부 조작 가능. 메뉴 토글은
  `aria-expanded`/`aria-controls` 갱신, 열림 시 첫 메뉴 링크로 포커스 이동, `Escape`로 닫힘 +
  포커스가 토글 버튼으로 복귀. 조치완료.
- **협력기관(네트워크) 배너 위치**: footer 전용 제약을 없앴다 — 이번 디자인은 기관소개·홈
  본문(`<main>`)에 배치한다. 여러 개의 `<nav>` 랜드마크(주 메뉴/푸터 메뉴, 소식 페이지는
  기관 분류 탭까지 3개)가 생겼지만 전부 서로 다른 `aria-label`을 가져 구분 가능 — 신규 게이트
  E4가 "nav가 있으면 각자 고유 라벨" 조건으로 이를 강제한다.
- **글자 배율(80~500%) 컨트롤**: 신규 기능. `<input type=range>` + `aria-label`(범위 명시) +
  `aria-live="polite"` 상태 문구로 변경 사실을 스크린리더에 알림. 값은 `localStorage`에
  저장(개인식별정보 아님, 단순 UI 선호값) — OWASP/KISA 관점에서 새로운 위험 없음.
- **"여기서 갈 수 있는 곳" 전이표 폐지**: 대신 header 주 메뉴(6항목) + footer 메뉴(4항목)가
  전 페이지 공통으로 항상 존재해 막다른 페이지를 구조적으로 막는다 — E7 게이트가 매 페이지
  주 메뉴·푸터 메뉴 존재를 확인.
- **h1 = title 페이지명 강제 폐지**: 홈·센터소개·전문상담·처음 오신 분께 페이지는 마케팅형
  hero 카피를 h1으로 쓴다(예: 홈 h1 = "상담과 정보를 제공하고..."). h1이 정확히 1개인 것만
  계속 검사(E5), 문자열 일치는 더 이상 강제하지 않음 — 독수리 승인 디자인의 의도적 선택.

### 전문가 심사 관점 — 기반은 갖춰짐
위 33개 항목 매핑 기준 22개 조치완료, 10개 해당없음(설계상 자연 충족), 1개는 GitHub에 위임(우리 통제 밖이지만 GitHub 자체가 성숙한 서비스). 자동화 검사(빌드 게이트 D1~D6, W3C validator, 명도 대비 계산)로 커버되는 항목은 실측까지 마쳤다.

### 지금 상태로 바로 신청하면 안 되는 이유 3가지

1. **사용자 심사가 아직 없음.** 인증 심사는 전문가 심사와 별개로 장애유형별(전맹·저시력, 상지지체, 청각) 실사용자의 실제 과업 수행 심사를 포함한다. 우리는 자동화 검사만 했을 뿐, 스펙 v1.3의 완료 기준(D9: "센스리더 실측 1회, 독수리 수동")을 아직 실행하지 않았다 — 이게 사실상 사용자 심사의 예행연습인데 건너뛴 상태.
2. **콘텐츠가 샘플/테스트 데이터 위주.** 소식 글 3건(그나마 2건은 "(샘플)" 표기), network-orgs 8곳뿐. 인증 심사는 실제 운영 콘텐츠를 대상으로 하므로, 실제 소식·기관소개 내용이 채워진 뒤 재점검이 필요하다.
3. **서류 요건 미확인.** 신청에 사업자등록증 사본이 필요하다(공식 안내 확인). 좋은비전의 사업자/법인 등록 여부는 이번 조사로 확인하지 못했다 — Unknown, 독수리 확인 필요.

### 결론
**기술적 기반은 신청 수준에 근접했으나, 지금 당장 신청 가능한 상태는 아니다.** 순서 제안: ① D9 센스리더 실측(무료, 독수리가 직접) → ② 실제 콘텐츠 채우기 → ③ 사업자등록 여부 확인 → ④ 그 다음 신청 검토. 비용은 페이지 수 구간에 따라 소(1~10p) 100만원부터(VAT 별도, 신규 기준) — 우리 사이트 규모(12페이지 안팎)면 최저 구간 해당 가능성이 높으나, 심사 기준의 "유형별 20페이지 이상" 요건이 이렇게 작은 사이트에 어떻게 적용되는지는 이번 조사로 확인하지 못함(불확실 — 기관에 직접 문의 필요).
