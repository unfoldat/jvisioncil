# 판정 — KWCAG 2.2 웹 접근성 매핑 + 인증 준비도

기록일: 2026-07-17 · 원문: 국가표준 KS X OT0003:2022 「한국형 웹 콘텐츠 접근성 지침 2.2」(4대 원칙·14개 지침·33개 검사항목, WCAG 2.1 기반 + 국내 추가 9개 항목) · 실측 기반, 짐작 없음

스펙 v1.3([base/스펙-사이트-v1.3.md](스펙-사이트-v1.3.md))에서 이미 설계한 것들(헤딩 위계, 랜드마크, 검색창 유일 편집창, 링크 텍스트 자족성, DOM=시각 순서)과 T7에서 실측한 것들(명도 대비, 포커스 표시, 터치 타겟)을 KWCAG 33개 항목에 매핑한다.

## 원칙 1. 인식의 용이성 (9개)

| 조항 | 항목명 | 우리 상태 | 근거 |
|---|---|---|---|
| 5.1.1 | 적절한 대체 텍스트 제공 | **조치완료** | `scripts/verify.mjs`가 alt 없는 `<img>` 발견 시 빌드 실패시킴(D6). CMS `body` 필드 hint에 "대체 텍스트를 반드시 채우세요" 명시 |
| 5.2.1 | 자막 제공 | **해당없음** | 멀티미디어(영상/오디오) 콘텐츠 자체가 없음. 재검토 트리거: 향후 영상 임베드(로드맵의 오프라인 연계 등) 추가 시 |
| 5.3.1 | 표의 구성 | **해당없음** | 사이트에 `<table>` 요소가 없음(전부 리스트/문단). 재검토 트리거: 표 콘텐츠 추가 시 |
| 5.3.2 | 콘텐츠의 선형구조 | **조치완료** | 스펙 §2.10 "DOM 순서 = 읽기 순서 = 시각 순서" — CSS `order`/`flex-direction: reverse` 등 시각만 바꾸는 규칙 미사용(코드 검토 확인) |

| 5.4.1 | 색에 무관한 콘텐츠 인식 | **조치완료** | 링크는 색+밑줄(`text-decoration: underline`) 이중 표시. `[공지]` 배지도 색이 아니라 텍스트로 구분 |
| 5.4.2 | 자동 재생 금지 | **해당없음** | 자동재생 미디어 없음. 자동 회전·캐러셀도 스펙 금지 목록에 이미 명시 |
| 5.4.3 | 텍스트 콘텐츠의 명도 대비 | **조치완료** | 실측(T7): 본문 텍스트 17.4:1, 링크 7.78:1, 포커스 링 4.60:1 — AA(4.5:1) 및 AAA(7:1)까지 충족 |
| 5.4.4 | 콘텐츠 간의 구분 | **조치완료** | 배지 테두리, footer 배경색 구분, 섹션 간 여백으로 시각적 구분 |

## 웹접근성 품질인증(WA 인증) 신청 가능 여부 평가

출처: 한국디지털접근성진흥원(kwacc.or.kr) 공식 안내 — 「디지털포용법」 제21조 근거, 심사기준 KWCAG 2.2, 서면심사 + 기술심사(전문가 심사 + 사용자 심사) 이원 구조, **33개 검사항목 평균 준수율 95% 이상**(전문가 심사) 합격 기준, 유효기간 1년.

### 전문가 심사 관점 — 기반은 갖춰짐
위 33개 항목 매핑 기준 22개 조치완료, 10개 해당없음(설계상 자연 충족), 1개는 GitHub에 위임(우리 통제 밖이지만 GitHub 자체가 성숙한 서비스). 자동화 검사(빌드 게이트 D1~D6, W3C validator, 명도 대비 계산)로 커버되는 항목은 실측까지 마쳤다.

### 지금 상태로 바로 신청하면 안 되는 이유 3가지

1. **사용자 심사가 아직 없음.** 인증 심사는 전문가 심사와 별개로 장애유형별(전맹·저시력, 상지지체, 청각) 실사용자의 실제 과업 수행 심사를 포함한다. 우리는 자동화 검사만 했을 뿐, 스펙 v1.3의 완료 기준(D9: "센스리더 실측 1회, 독수리 수동")을 아직 실행하지 않았다 — 이게 사실상 사용자 심사의 예행연습인데 건너뛴 상태.
2. **콘텐츠가 샘플/테스트 데이터 위주.** 소식 글 3건(그나마 2건은 "(샘플)" 표기), network-orgs 8곳뿐. 인증 심사는 실제 운영 콘텐츠를 대상으로 하므로, 실제 소식·기관소개 내용이 채워진 뒤 재점검이 필요하다.
3. **서류 요건 미확인.** 신청에 사업자등록증 사본이 필요하다(공식 안내 확인). 좋은비전의 사업자/법인 등록 여부는 이번 조사로 확인하지 못했다 — Unknown, 독수리 확인 필요.

### 결론
**기술적 기반은 신청 수준에 근접했으나, 지금 당장 신청 가능한 상태는 아니다.** 순서 제안: ① D9 센스리더 실측(무료, 독수리가 직접) → ② 실제 콘텐츠 채우기 → ③ 사업자등록 여부 확인 → ④ 그 다음 신청 검토. 비용은 페이지 수 구간에 따라 소(1~10p) 100만원부터(VAT 별도, 신규 기준) — 우리 사이트 규모(12페이지 안팎)면 최저 구간 해당 가능성이 높으나, 심사 기준의 "유형별 20페이지 이상" 요건이 이렇게 작은 사이트에 어떻게 적용되는지는 이번 조사로 확인하지 못함(불확실 — 기관에 직접 문의 필요).
