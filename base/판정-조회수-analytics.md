# 판정 — 조회수 표시 안 만듦, Cloudflare Web Analytics로 대체

기록일: 2026-07-17 · 독수리 지정

## 결정

**개별 소식 글에 "조회수 N" 같은 표시를 만들지 않는다.** 사이트 전체 트래픽 확인은 Cloudflare Web Analytics(대시보드)로 한다.

## 이유

- 조회수 카운터 = 방문 기록을 세고 보관하는 기능. 스펙 §0.1 무보관 원칙("이용자 데이터를 사이트에 저장하지 않는다")과 정면으로 충돌한다.
- 서버·DB 없는 정적 사이트 구조(§5 금지 목록 "DB·서버 기능 도입 금지")에서 조회수를 구현하려면 결국 외부 저장소나 KV가 필요해진다 — 사이트 성격("데이터를 보관하지 않고 연결한다")에 맞지 않는 기능을 위해 예외를 만드는 것.
- Cloudflare Web Analytics는 쿠키·localStorage를 쓰지 않는 방식으로 동작해 금지 목록("localStorage/sessionStorage/쿠키 사용 금지")과 충돌하지 않는다. 집계는 Cloudflare 쪽에서 처리되고, 사이트 코드에는 저장 기능이 남지 않는다.
- "몇 명이 봤는지"는 방문자(당사자·가족·외부 지원자)에게 노출할 정보가 아니라 운영 참고용 정보 — 페이지에 표시할 이유가 없다.

## 남은 일 (선택, 대시보드)

Cloudflare Web Analytics를 켜려면 Cloudflare 대시보드에서 사이트 등록 후 발급되는 beacon 스크립트를 넣어야 한다. 지금은 미적용 — 필요해지면 별도 요청으로 진행 (스크립트 삽입 1줄, 코드 구조 변경 없음).
